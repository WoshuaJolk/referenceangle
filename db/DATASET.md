# referenceangle face dataset + Postgres seed

A redistributable single-face portrait dataset for reviving **referenceangle.com**.
It produces cropped portrait JPGs plus a `seed.sql` that fills the app's
`reference` table with per-image head pose and facial attributes.

## Artifacts

| Path | What |
|------|------|
| `images/NNNNN.jpg` | Square 256x256 JPG (q82), one clear face, center-cropped. **5,000 images, ~65 MB total.** |
| `seed.sql` | `CREATE TABLE reference (...)` + one `INSERT` per image (5,000 rows). |
| `LICENSES.csv` | Per-image attribution: filename, source dataset URL, source ref, license, license URL, author. |
| `stats.json` | Final counts + per-attribute distribution (gender, emotions, boolean fractions, yaw/pitch/roll ranges + 10-degree histograms). |
| `build_from_sfhq.py` | The pipeline (streams source, estimates attributes, writes all artifacts). |
| `attrs.py` | Attribute estimation module (pose / blendshapes / DeepFace / heuristics). |
| `models/face_landmarker.task` | MediaPipe FaceLandmarker model (identical file to the sibling `stare` repo's). |

## Image source + license

**SFHQ-T2I — Synthetic Faces HQ, text-to-image subset**, streamed from the
Hugging Face Hub dataset [`bitmind/SyntheticFacesHQ`](https://huggingface.co/datasets/bitmind/SyntheticFacesHQ)
(file `SyntheticFacesHighQuality-T2I.zip`, ~122k 1024x1024 images generated by
diffusion models — Flux / DALL-E 3 / SDXL). The SFHQ dataset is MIT-licensed and,
because every image is **synthetically generated**, there are **no privacy,
personality, or subject-image-rights issues** — it is freely redistributable in
a public GitHub repo. `LICENSES.csv` records the source dataset URL, the SFHQ
license URL, and the author (David Beniaguev / SelfishGene) for each image.

Why SFHQ instead of Wikimedia Commons / Unsplash: Commons portrait thumbnails are
served from `upload.wikimedia.org`, which hard rate-limits bulk image downloads
(HTTP 429, ~1 image/sec sustained per IP) — impractical for 5,000 images. The
Unsplash Lite dataset needs a manual agreement/download. SFHQ streams from the HF
CDN (a single ~23 GB zip, unzipped sequentially in-flight) at CDN bandwidth, and
synthetic faces carry the cleanest possible redistribution license.

The earlier Wikimedia-Commons pipeline (recursive CC0/PD/CC-BY category harvest +
rate-limited downloader) is preserved in `harvest*.py` / `process_dataset.py` for
reference, but was abandoned for the source above due to the rate limit.

## Pipeline (`build_from_sfhq.py`)

1. **Stream**: one HTTP GET of the T2I zip, unzipped sequentially in-flight with
   `stream-unzip` (~40 img/s, CDN-bound). The stream is **resumable** — on a
   dropped connection it reconnects with an HTTP `Range` request and continues.
   Streaming runs in a separate lightweight process (no TensorFlow) feeding a
   bounded queue, so it never starves the CPU work.
2. **Detect / crop**: downscale to ~560px, run **MediaPipe Tasks FaceLandmarker**;
   require exactly one **dominant** face (reject group shots where a 2nd face is
   >30% of the largest); center-crop a square (1.6x margin), resize to 256.
3. **Estimate** all attributes (below).
4. **Yaw-balance**: the synthetic pool is frontal-heavy, so near-frontal poses
   (|yaw|<3, and some |yaw|<8) are probabilistically down-sampled to widen the
   final yaw distribution.
5. **Write** the image, append `LICENSES.csv`, collect the SQL row; checkpoint
   `seed.sql` / `stats.json` every 250 kept images.

Runs as a single CPU process with one TensorFlow instance — deliberately, because
this is a RAM-constrained shared machine where multiple TF workers thrash swap.
Throughput ~1.8 kept images/sec; 5,000 images in ~48 min.

## Attribute estimation (open-source models)

- **Head pose `pitch` / `yaw` / `roll` (degrees)** — from the MediaPipe
  FaceLandmarker **facial transformation matrix** of the dominant face,
  decomposed with the **exact `head_pose_from_matrix` function copied verbatim
  from the sibling `stare` repo** (`stare/pipeline/features.py`). The Python
  Tasks API matrix is passed straight through (`np.asarray(matrix)`), matching
  `stare/pipeline/encode.py`, so angles share stare's convention bit-for-bit:
  **yaw+ = head turned right, pitch+ = up, roll+ = tilted right**. No extra
  negation is applied (the frontend calibrates to this convention separately).
  Images with no transformation matrix are skipped.
- **Age / gender / emotion** — **DeepFace** `analyze` (one call). Age becomes a
  `±4` bracket (`lowage`/`highage`). Gender → `Male`/`Female`. Emotion mapped to
  the app's labels and stored UPPERCASE:

  | DeepFace | stored |   | DeepFace | stored |
  |----------|--------|---|----------|--------|
  | happy | HAPPY |   | disgust | DISGUSTED |
  | sad | SAD |   | surprise | SURPRISED |
  | angry | ANGRY |   | neutral | CALM |
  | fear | CONFUSED |   | | |

- **isSmiling / mouthOpen / eyesOpen** — from FaceLandmarker **blendshapes**
  (`mouthSmileLeft/Right`, `jawOpen`, `eyeBlinkLeft/Right`). Smiling correlates
  with HAPPY but is computed independently.
- **eyeGlasses / sunGlasses / beard / mustache** — landmark-region image
  heuristics (Canny edge density across the eye band for glasses; brightness +
  edges for sunglasses; lower-face vs cheek darkness/texture for beard; nose-to-
  lip band texture for mustache). **Approximate**, but give a varied, every-
  filter-returns-results distribution.

## `reference` table schema

```sql
CREATE TABLE reference (
  url text,                 -- '/images/NNNNN.jpg'
  pitch numeric, yaw numeric, roll numeric,   -- degrees, stare convention
  gender text,              -- 'Male' | 'Female'
  lowage int, highage int,  -- age bracket (est ±4)
  emotions text,            -- UPPERCASE, e.g. 'HAPPY'
  issmiling boolean, sunglasses boolean, eyeglasses boolean,
  beard boolean, mustache boolean, eyesopen boolean, mouthopen boolean
);
```
Lowercase unquoted identifiers (Postgres folds them); the frontend query uses
`strpos(emotions, UPPER(x))` and negates yaw on its side.

## Final counts & distribution (`stats.json`)

- **Count: 5,000** (from 8,848 candidates processed; rejections: 3,396 yaw-balance
  down-samples, 345 no-face, 39 multi-face, 37 too-small, 31 mesh-fail).
- **Gender**: Female 2,502 / Male 2,498 (balanced).
- **Emotions** (all 7 present): CALM 1,683, SAD 1,150, HAPPY 941, CONFUSED 585,
  ANGRY 558, SURPRISED 80, DISGUSTED 3.
- **Boolean true-fractions**: eyeglasses 0.47, mustache 0.62, beard 0.42,
  isSmiling 0.19, sunglasses 0.09, mouthOpen 0.055, eyesOpen 0.98 (so ~80
  closed-eye, ~275 mouth-open, ~465 sunglasses, etc. — every filter returns rows).
- **Yaw**: range -39.2 .. +43.1; concentrated near 0 with tails to ±40 (synthetic
  faces are frontal-biased; yaw-balancing widened the spread).
- **Pitch**: range -64.4 .. +67.5 (well spread). **Roll**: -35.4 .. +32.6.

## Environment / reproduce

Python 3.12 venv (`venv/`) with `mediapipe` (Tasks API), `deepface` +
`tensorflow`/`tf-keras`, `datasets`, `stream-unzip`, `opencv-python`, `pillow`,
`numpy`, `requests`. CPU only.

```bash
TARGET=5000 WORKERS=1 python build_from_sfhq.py   # streams source, writes images/, seed.sql, LICENSES.csv, stats.json
```
